# 线程池（C语言版）

## 原理：
创建并管理多个线程，重复使用来执行任务，从而减少线程创建销毁所带来的开销
------------------------------------

## 核心部分：
1. 任务队列：存储需要处理的任务，由工作的线程来处理
    * 任务队列中的内容是一些处理的函数（流程），因此任务队列中存储的任务函数的地址
	* 通过线程池提供的API来进行任务的添加与删除
	* 调用线程函数往任务队列中添加任务的线程就是生产者线程

2. 工作的线程：任务队列的消费者，有任意个
    * 线程池中需要维护一定的工作线程的数量，用以不停地处理任务队列中的任务
	* 任务队列为空时阻塞，有新任务时会继续工作

3. 管理者线程：负责管理工作的线程
    * 不处理任务队列中任务的处理
	* 任务是周期性的检测任务队列的任务数量和处于工作状态的线程数量
	* 任务过多时，创建一些新的工作线程
	* 任务过少时，删除一些空闲的工作线程
